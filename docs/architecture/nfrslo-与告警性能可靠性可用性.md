# NFR/SLO 与告警（性能/可靠性/可用性）

## 目标（初版）
- API：GET P95 ≤ 300ms、P99 ≤ 800ms；写操作 P95 ≤ 800ms
- 作业：导入/抽取/差异 Job 启动延迟 P95 ≤ 1s；处理时长按批量规模约束并报出 ETA
- 队列：滞留时间 P95 ≤ 10s；失败率 ≤ 1%（自动重试 3 次）
- 可用性：月度 API 可用性 ≥ 99.5%

## 告警建议
- 错误率 > 2%（5 分钟窗口）
- API 延迟 P95 > 800ms（连续 5 个窗口）
- 队列滞留 > 60s 或 DLQ 积压
- 导出任务 > 阈值（并发/时长）
