## Technique Sessions

### What If Scenarios - 热身（约10分钟）
**Description:** 通过连续的“如果……会怎样？”问题，快速激发思维，探索跨DOORS→系统需求→指标→RBOM→下游设计全链路下的潜在新可能。

**Ideas Generated:**
1. 对现有工作流程的影响
2. 对现有数据的影响
3. 如何快速分析出影响范围，并通知相关人员进行动态调整
4. 字段的自动映射
5. 参数的自动提取
6. 与RBOM节点的自动关联
7. 变更包自动生成与分发
8. 自动生成更改单
9. 基于更改单进行结构化影响分析，形成待办并可触发/影响任务

**Insights Discovered:**
- 核心关注集中在“影响分析与联动响应”（流程/数据/通知）
- 需要自动化的变更溯源与范围评估能力
- 自动化主链路清晰：字段映射 → 参数/指标提取 → RBOM节点关联
- 以上能力一旦贯通，可为差异影响报告与动态协同提供基础
- 差异报告应直接派生变更单与任务编排，缩短响应闭环

**Notable Connections:**
- 上游变更 → 影响范围分析 → 通知/协同 → 下游方案调整的闭环
- 与RBOM差异分析、需求模板映射能力存在天然联动
- 模板映射 → 指标抽取 → RBOM节点绑定，形成数据驱动的设计联动
- RBOM差异 → 变更单/变更包 → 待办流水线/工单系统 → 跨团队协同

### Mind Mapping - 发散（约20–30分钟）
**Description:** 以“需求集成”为中心，围绕上游需求、系统需求模板与指标、RBOM、下游专业、变更与验证、数据治理与合规等，发散主要分支与子分支，追求广度与关联性。

**Ideas Generated:**
- 分支1：上游需求源与同步（DOORS字段、变更订阅、溯源规则）
  - A 必须抽取字段/元数据：唯一标记、编号、标题、内容、验证方式、优先级
  - B 同步策略：手动（按需触发）
  - C 溯源唯一键：模块ID、对象ID、基线ID
 - 分支2：系统需求模板与映射（字段映射、口径校验、去重逻辑）
  - A 模板字段集合：系统需求ID、来源需求引用、标题、内容、指标、验证方式、优先级、状态、可行性、可行性分析、验证活动
  - B 映射与口径规则：英文匹配、中文匹配、统一参数描述、人为解决冲突
  - C 去重与冲突处理：来源ID、人工复核

- 分支3：指标/参数抽取与口径管理（单位/量纲、阈值、版本）
  - A 指标来源与抽取规则：标题/内容解析、DOORS结构化字段、命名约定、上下文/单位识别、附件表格解析、正则/规则库、词典/MDM、人工补录
  - B 口径与单位/量纲规范：单位统一与换算、精度与小数规则、范围/阈值表达（含上下限与闭开区间）、取值域/枚举、默认值与缺省口径
  - C 版本与变更管理：指标版本/基线策略、变更比较与兼容（废弃/替代/映射）、历史追溯与审计

 - 分支4：RBOM结构与关联策略（零部件/功能项/接口的绑定与粒度）
  - A RBOM层级与粒度：多维管理=是（功能/物料/接口）；层级=待确认
  - B 关联绑定策略：直接绑定Part；允许多对多/继承/聚合（待细化）
  - C 更新与差异跟踪：RBOM版本/基线策略；指标或需求变更时的重连/级联策略；差异报告字段=变更类型、影响对象、影响级别、责任人

 - 分支5：变更单/变更包与任务编排（自动派生、分发、责任人、SLA）
  - A 字段集合：标题、变更类型、来源需求/指标、受影响对象、影响级别、责任人、截止期、关联任务/评审、状态、备注
  - B 任务编排与分发：根据工作包绑定的需求进行分析并定位当时责任人进行分发；消息合并由人员统一发送；一条需求可对应多个任务；一个消息可对应多个任务；不需要对接其他工具
 - C SLA/优先级/升级：无需该特性（当前阶段）

 - 分支6：影响范围分析（依赖图、影响评分、跨专业）
  - A 依赖图与对象关系：需求→系统需求→指标→RBOM节点→设计任务→验证用例/仿真工况→结果数据（参数/报告/模型等）
  - B 影响评分类目与算法：暂不需要（当前阶段）
 - C 跨专业提示与路由：触发规则=必触发；路由对象=对应责任人；提示渠道=站内信及OA

 - 分支7：下游专业协同（方案/详设/仿真/验证的串并行与里程碑）
  - A 专业链路与里程碑：方案设计→详细设计→仿真→样机/试制→验证/试验→发布；完成判定=评审通过
  - B 协同节奏与并行限制：验证需依赖RBOM基线
 - C 变更对策与回溯：依赖人为判断，提供严格而准确的手动回滚机制

**Insights Discovered:**
- （待补充）
 - 模板字段覆盖从系统需求到验证活动的关键生命周期要素，有利于端到端追踪

- 分支8：验证与回归（用例更新、仿真工况、回归计划与看板）
  - A 用例/工况更新规则：变更影响≥中→相关用例自动标记失效并生成更新任务；指标变更→仿真工况自动重算边界；验证活动与系统需求/指标双向追溯
  - B 回归计划与看板：按变更包生成回归批次；按RBOM层级分组执行；看板展示通过率/阻塞项/剩余工作量
  - C 产出物与留痕：报告、数据集、仿真模型、日志与审计链；版本签名与审批记录；对外发布的可复现包

## Idea Categorization

### Immediate Opportunities
1. 系统需求模板与映射：确定字段集与中英映射、统一参数描述与去重键（来源ID+标题/内容哈希）
2. 指标抽取MVP：规则库+正则+词典/MDM与单位/量纲换算；人工复核兜底
3. RBOM关联MVP：直接绑定Part，建立RBOM基线与差异报告字段（变更类型/对象/级别/责任人）
4. 变更编排闭环：差异→变更包/更改单→任务分发（按责任人）→站内信/OA通知→人工严格回滚机制
5. 验证与回归：变更≥中自动使例失效并派生更新任务；按变更包生成回归批次与RBOM层级分组的看板
6. 影响路由：“必触发+责任人”最小策略先落地

### Future Innovations
1. 多维RBOM（功能/接口/物料）与多对多/继承/聚合权重策略
2. 自动化影响评分（幅度/覆盖/关键度/成熟度）与等级映射
3. 基线/版本自动同步与对比、接口冻结策略辅助
4. 与外部工单/看板系统的松耦合集成（保留接口，不立即对接）
5. 指标抽取半自动学习：基于历史修订形成可扩展规则库

### Moonshots
1. 基于NLP/LLM的中英双语字段映射与指标抽取高置信
2. 数字线程：指标变化驱动仿真边界与验证数据自动更新与触发
3. 组织级度量自动化（需求成熟度、吞吐、响应/修复时长等）
